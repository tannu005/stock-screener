# Deployment Guide - Stock Screener Pro\n\nThis guide covers deploying Stock Screener Pro to production with live market data.\n\n## Vercel Deployment (Recommended)\n\nVercel is optimized for Next.js and requires minimal configuration.\n\n### Prerequisites\n\n- GitHub account with stock-screener repo pushed\n- Vercel account (free at vercel.com)\n- API key from financial provider (Finnhub, Alpha Vantage, etc.)\n\n### Step 1: Connect Repository\n\n1. Go to https://vercel.com\n2. Click \"New Project\"\n3. Select \"Import Git Repository\"\n4. Search and select `stock-screener`\n5. Click \"Import\"\n\n### Step 2: Configure Environment Variables\n\n1. In Vercel dashboard, go to Project Settings → Environment Variables\n2. Add the following variables:\n\n```\nName: NEXT_PUBLIC_DATA_MODE\nValue: live\nEnvironments: Production, Preview, Development\n\nName: FINNHUB_API_KEY\nValue: [your_finnhub_api_key]\nEnvironments: Production, Preview, Development\n\nName: NEXT_PUBLIC_APP_ENVIRONMENT\nValue: production\nEnvironments: Production\n```\n\n**Note:** Variables prefixed with `NEXT_PUBLIC_` are exposed to the frontend (safe).\nVariables without prefix are server-only (secure for API keys).\n\n### Step 3: Deploy\n\n1. Click \"Deploy\"\n2. Wait for build to complete (2-3 minutes)\n3. Click \"Visit\" to open deployed app\n4. Verify real data is showing\n\n### Step 4: Monitor Deployment\n\nVercel automatically redeploys on:\n- Push to main branch\n- Pull requests (preview deployments)\n\nView logs and analytics:\n- Deployments tab: See build logs\n- Analytics tab: Monitor real-time usage\n- Functions: Check API route execution\n\n## Environment Variables by Environment\n\n### Development (.env.local)\n\nUse simulated data for fast iteration:\n\n```\nNEXT_PUBLIC_DATA_MODE=simulated\n# No API key needed\n```\n\n### Staging (Vercel Preview)\n\nTest with real data using hybrid mode:\n\n```\nNEXT_PUBLIC_DATA_MODE=hybrid\nFINNHUB_API_KEY=[staging_key]\nNEXT_PUBLIC_APP_ENVIRONMENT=staging\n```\n\n### Production (Vercel Main)\n\nFull real-time data:\n\n```\nNEXT_PUBLIC_DATA_MODE=live\nFINNHUB_API_KEY=[production_key]\nNEXT_PUBLIC_APP_ENVIRONMENT=production\nAPI_CACHE_DURATION=600000\nMAX_STOCKS_PER_REQUEST=50\n```\n\n## API Rate Limiting in Production\n\n### Recommended Settings for 5000 Stocks\n\n**Finnhub Free Tier (60 calls/min):**\n```\nNEXT_PUBLIC_DATA_MODE=hybrid  # Only fetch top 20 live\nAPI_CACHE_DURATION=600000     # 10 minute cache\nMAX_STOCKS_PER_REQUEST=20\n```\n\n**Finnhub Pro (500 calls/min):**\n```\nNEXT_PUBLIC_DATA_MODE=live\nAPI_CACHE_DURATION=300000     # 5 minute cache\nMAX_STOCKS_PER_REQUEST=100\n```\n\n### Monitoring Rate Limits\n\n1. In Vercel Functions tab, monitor API route duration\n2. Check error logs for rate limit messages\n3. Adjust `API_CACHE_DURATION` if seeing 429 errors\n\n## Alternative Deployment Platforms\n\n### Netlify\n\n1. Connect GitHub repo\n2. Build command: `npm run build`\n3. Publish directory: `.next`\n4. Add environment variables in Site Settings\n5. Deploy\n\n**Note:** Netlify requires paid tier for background functions if using WebSocket server.\n\n### Railway.app\n\n1. Connect GitHub repo\n2. Railway auto-detects Next.js\n3. Add environment variables\n4. Deploy\n5. Available at railway.app domain\n\n### Docker Deployment\n\nCreate `Dockerfile`:\n\n```dockerfile\nFROM node:18-alpine\n\nWORKDIR /app\n\nCOPY package*.json ./\nRUN npm ci --only=production\n\nCOPY . .\nRUN npm run build\n\nEXPOSE 3000\n\nCMD [\"npm\", \"start\"]\n```\n\nDeploy:\n\n```bash\n# Build image\ndocker build -t stock-screener .\n\n# Run container\ndocker run -p 3000:3000 \\\n  -e FINNHUB_API_KEY=your_key \\\n  -e NEXT_PUBLIC_DATA_MODE=live \\\n  stock-screener\n```\n\n## Performance Optimization\n\n### Image Optimization\n\nVercel automatically optimizes images. Ensure:\n\n1. Use `next/image` for all stock logos\n2. Optimize company profile images\n3. Check Vercel Analytics for image delivery times\n\n### Database Caching (Optional)\n\nFor higher traffic, add database-level caching:\n\n1. Use Vercel KV (Redis)\n2. Cache popular stock queries\n3. Extend cache TTL for stable stocks\n\n```typescript\n// Example: Cache with Vercel KV\nimport { kv } from '@vercel/kv';\n\nconst cached = await kv.get(`stock:${symbol}`);\nif (cached) return JSON.parse(cached);\n```\n\n### CDN Configuration\n\nVercel automatically uses CDN for:\n- Static assets (CSS, JS, images)\n- API responses (with Cache-Control headers)\n- ISR (Incremental Static Regeneration)\n\nCurrent API routes use:\n```\nCache-Control: public, s-maxage=60, stale-while-revalidate=300\n```\n\nThis means:\n- Edge cache: 60 seconds\n- Stale cache: 300 seconds while revalidating\n- Reduced API calls by ~83%\n\n## Monitoring & Analytics\n\n### Key Metrics to Track\n\n1. **Function Duration** - API response times\n2. **Error Rate** - Failed API calls\n3. **Cache Hit Rate** - Reduction in API calls\n4. **Deployment Frequency** - How often you deploy\n5. **Data Freshness** - Cache age for stock prices\n\n### Vercel Analytics\n\nEnable in Project Settings → Analytics:\n\n1. Web Analytics (free) - Page performance\n2. Real Experience Score (free) - Core Web Vitals\n3. Custom Metrics (pro) - Custom monitoring\n\n### Error Tracking\n\nEnable error tracking:\n\n1. Check Vercel Logs\n2. Look for API errors in Functions tab\n3. Monitor rate limit errors (HTTP 429)\n4. Check for timeout errors (>60s)\n\n## Scaling Considerations\n\n### Current Limits\n\nWith Finnhub free tier:\n- 5,000 stocks × 60 calls/min = ~12 stocks/min fetch rate\n- 5-minute cache = ~300 concurrent users on shared stocks\n- Recommended: Hybrid mode for 5000+ stocks\n\n### Upgrade Path\n\n**If scaling to 100,000+ stocks:**\n\n1. Upgrade Finnhub (500+ calls/min)\n2. Add Redis caching (Vercel KV)\n3. Implement background job (Vercel Cron)\n4. Use incremental fetch strategy\n\n**If scaling to 1M+ requests/day:**\n\n1. Switch to dedicated API provider\n2. Set up dedicated backend service\n3. Implement distributed caching\n4. Use message queues for updates\n\n## Troubleshooting Deployment\n\n### Build Fails\n\n**Error: \"Module not found\"**\n```bash\n# Solution: Clear dependencies\nrm -rf node_modules package-lock.json\nnpm install\ngit push  # Redeploy\n```\n\n**Error: \"ENOENT: no such file\"**\n```\nMissing environment variables\nCheck .env.local is NOT committed\nAdd all NEXT_PUBLIC_ vars to Vercel\n```\n\n### Runtime Errors\n\n**\"API Key not configured\"**\n- Check Environment Variables in Vercel dashboard\n- Ensure exact variable names match\n- Restart deployment after adding vars\n\n**\"Rate limit exceeded\"**\n- Increase cache duration\n- Switch to hybrid mode\n- Upgrade API plan\n- Implement request batching\n\n### Performance Issues\n\n**Slow page load**\n- Check Vercel Analytics\n- Reduce initial stock count\n- Use hybrid mode\n- Enable compression (automatic)\n\n## Security Best Practices\n\n1. **API Keys**\n   - Never commit .env.local\n   - Use Vercel Environment Variables\n   - Rotate keys periodically\n   - Use separate keys per environment\n\n2. **Rate Limiting**\n   - Implement on API routes\n   - Monitor for abuse\n   - Cache aggressively\n   - Batch requests\n\n3. **Data Validation**\n   - Validate API responses\n   - Type-check all data\n   - Handle missing fields gracefully\n   - Log errors securely\n\n4. **HTTPS**\n   - Automatically enabled on Vercel\n   - All connections encrypted\n   - Subdomains available\n\n## Rollback Procedure\n\n1. In Vercel dashboard → Deployments\n2. Find previous stable deployment\n3. Click menu → Promote to Production\n4. Wait for rollback to complete\n5. Verify old version is live\n\n## Support & Resources\n\n- [Vercel Docs](https://vercel.com/docs)\n- [Next.js Deployment](https://nextjs.org/docs/deployment)\n- [Finnhub API Limits](https://finnhub.io/docs/api/rate-limits)\n- [Stock Screener Pro GitHub](https://github.com/tannu005/stock-screener)\n
