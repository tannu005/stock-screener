# Live Market Data Integration\n\nThis document describes the Stock Screener Pro's live market data integration system, including setup instructions for connecting real financial APIs.\n\n## Overview\n\nStock Screener Pro supports three data modes:\n\n- **`simulated`** (default) - Uses generated stock data. No API keys required. Best for development/demo.\n- **`live`** - Fetches real-time data from configured financial API. Requires valid API key.\n- **`hybrid`** - Combines live data for popular stocks with simulated data for padding. Balanced approach.\n\n## Architecture\n\n### Components\n\n```\n┌─────────────────────────────────────────────────────────────┐\n│                     React Frontend                          │\n│  (useWebSocket hook, StockTable, HeroHeader, etc.)          │\n└────────────────────┬────────────────────────────────────────┘\n                     │\n                     │ HTTP/WebSocket\n                     ↓\n┌─────────────────────────────────────────────────────────────┐\n│            Next.js API Routes (/app/api/stocks/)            │\n│  ├─ /quote - Get single or multiple stock quotes            │\n│  ├─ /search - Search for stocks                            │\n│  └─ /historical - Fetch historical price data              │\n└────────────────────┬────────────────────────────────────────┘\n                     │\n                     ↓\n┌─────────────────────────────────────────────────────────────┐\n│           Market Data Service Layer                         │\n│  (src/lib/api/marketData.ts)                               │\n│  ├─ Handles provider selection                             │\n│  ├─ Implements fallback logic                              │\n│  ├─ Manages caching                                        │\n│  └─ Handles rate limiting                                  │\n└────────────────────┬────────────────────────────────────────┘\n                     │\n      ┌──────────────┼──────────────┐\n      ↓              ↓              ↓\n┌──────────┐   ┌──────────┐   ┌──────────┐\n│ Finnhub  │   │  Alpha   │   │ Polygon  │\n│   API    │   │ Vantage  │   │   API    │\n└──────────┘   └──────────┘   └──────────┘\n```\n\n## Setup Instructions\n\n### 1. Choose a Financial API Provider\n\n#### Option A: Finnhub (Recommended)\n\nFinnhub offers real-time market data with a generous free tier:\n\n1. Go to [https://finnhub.io](https://finnhub.io)\n2. Sign up for a free account\n3. Generate an API key from your dashboard\n4. Add to `.env.local`:\n   ```\n   FINNHUB_API_KEY=your_api_key_here\n   ```\n\n**Advantages:**\n- Real-time WebSocket support\n- Generous free tier (60 API calls/min)\n- Company profiles, technical indicators\n- No commercial restrictions for free tier\n\n**Rate Limits:**\n- Free tier: 60 API calls/minute\n- Premium tier: Starting at 500+ calls/minute\n\n#### Option B: Alpha Vantage\n\nAlpha Vantage provides free stock market data with simple REST API:\n\n1. Go to [https://www.alphavantage.co](https://www.alphavantage.co)\n2. Sign up for a free API key\n3. Add to `.env.local`:\n   ```\n   ALPHA_VANTAGE_API_KEY=your_api_key_here\n   ```\n\n**Advantages:**\n- Completely free\n- Simple REST API\n- No registration required beyond email\n- Technical indicators included\n\n**Limitations:**\n- Rate limit: 5 API calls/minute (free tier)\n- Slower response times\n- Limited historical data\n\n#### Option C: Polygon.io\n\nPolygon provides enterprise-grade market data:\n\n1. Go to [https://polygon.io](https://polygon.io)\n2. Sign up for a free tier account\n3. Generate API key from dashboard\n4. Add to `.env.local`:\n   ```\n   POLYGON_API_KEY=your_api_key_here\n   ```\n\n**Advantages:**\n- High-quality data\n- Excellent documentation\n- Crypto, forex, stocks unified\n- Free tier available\n\n**Limitations:**\n- Free tier limited to end-of-day data\n- Requires paid subscription for intraday\n\n#### Option D: IEX Cloud\n\nIEX Cloud offers comprehensive market data:\n\n1. Go to [https://iexcloud.io](https://iexcloud.io)\n2. Create account\n3. Generate publishable key\n4. Add to `.env.local`:\n   ```\n   IEX_CLOUD_API_KEY=your_api_key_here\n   ```\n\n**Advantages:**\n- Reliable, enterprise-grade\n- Fast API responses\n- Rich financial data\n- Free tier available\n\n**Limitations:**\n- Free tier has limited message allowance\n- Requires paid subscription for full features\n\n### 2. Configure Environment Variables\n\nCreate or update `.env.local` in the project root:\n\n```bash\n# Data Mode\nNEXT_PUBLIC_DATA_MODE=live  # or 'simulated' or 'hybrid'\n\n# Choose ONE primary API provider\nFINNHUB_API_KEY=your_key_here\n# ALPHA_VANTAGE_API_KEY=your_key_here\n# POLYGON_API_KEY=your_key_here\n# IEX_CLOUD_API_KEY=your_key_here\n\n# Cache configuration\nAPI_CACHE_DURATION=300000  # 5 minutes in milliseconds\nMAX_STOCKS_PER_REQUEST=100\n\n# Feature flags\nNEXT_PUBLIC_ENABLE_LIVE_UPDATES=true\nNEXT_PUBLIC_ENABLE_RATE_LIMIT_PROTECTION=true\n```\n\n### 3. Install Dependencies\n\n```bash\nnpm install\n# or\nyarn install\n```\n\nKey dependencies for live data:\n- `axios` - HTTP client for API requests\n- `ws` - WebSocket support\n- `node-cache` - Server-side caching\n\n### 4. Start Development Server\n\n```bash\nnpm run dev\n```\n\nThe application will automatically:\n1. Detect `NEXT_PUBLIC_DATA_MODE` setting\n2. Load data based on configured mode\n3. Fall back to simulated data if API fails\n4. Cache API responses to reduce rate limits\n\n## Usage Examples\n\n### Using the Market Data Service\n\n```typescript\n// In a Next.js API route or server component\nimport { marketDataService } from '@/lib/api/marketData';\n\n// Get single stock\nconst stock = await marketDataService.getStockQuote('AAPL');\n\n// Get multiple stocks\nconst stocks = await marketDataService.getMultipleStocks(['AAPL', 'MSFT', 'GOOGL']);\n```\n\n### Using the Stock Data Loader\n\n```typescript\n// In components or API routes\nimport { stockDataLoader } from '@/lib/data/stockDataLoader';\n\n// Load stocks based on configured mode\nconst stocks = await stockDataLoader.loadStocks(5000);\n\n// Update a single stock\nconst updated = await stockDataLoader.updateStock(stock);\n\n// Watch a stock for real-time updates\nconst unwatch = stockDataLoader.watchStock('AAPL', (stock) => {\n  console.log('Stock updated:', stock);\n});\n\n// Stop watching\nunwatch();\n```\n\n### API Endpoints\n\n#### Get Stock Quote\n\n```bash\n# Single stock\nGET /api/stocks/quote?symbol=AAPL\n\n# Multiple stocks\nGET /api/stocks/quote?symbols=AAPL,MSFT,GOOGL\n```\n\nResponse:\n```json\n{\n  \"data\": [\n    {\n      \"symbol\": \"AAPL\",\n      \"name\": \"Apple Inc.\",\n      \"price\": 195.50,\n      \"change\": 2.50,\n      \"changePct\": 1.29,\n      ...\n    }\n  ]\n}\n```\n\n#### Search Stocks\n\n```bash\nGET /api/stocks/search?q=apple&limit=10\n```\n\n## Data Modes Deep Dive\n\n### Simulated Mode\n\n**Best for:** Development, demos, testing, screenshots\n\n```javascript\n// .env.local\nNEXT_PUBLIC_DATA_MODE=simulated\n```\n\nFeatures:\n- No API keys required\n- Deterministic data (same every run)\n- 5000 stocks with full technical indicators\n- Instant startup (no API delays)\n- Realistic market data patterns\n\n### Live Mode\n\n**Best for:** Production, real market data analysis\n\n```javascript\n// .env.local\nNEXT_PUBLIC_DATA_MODE=live\nFINNHUB_API_KEY=your_key\n```\n\nFeatures:\n- Real-time market data\n- Live price updates\n- Actual company information\n- Limited by API rate limits\n- May have startup delays\n- Fallback to simulated if API fails\n\n### Hybrid Mode\n\n**Best for:** Balance between real data and performance\n\n```javascript\n// .env.local\nNEXT_PUBLIC_DATA_MODE=hybrid\nFINNHUB_API_KEY=your_key\n```\n\nFeatures:\n- Real data for top 20 stocks\n- Simulated data for remaining stocks\n- Fast startup with real popular stocks\n- Reduced API rate limit usage\n- All premium UI features intact\n\n## Rate Limiting & Optimization\n\n### Built-in Rate Limiting\n\nThe market data service includes:\n\n1. **Response Caching**\n   - Default: 5-minute TTL\n   - Configurable via `API_CACHE_DURATION`\n   - Reduces API calls significantly\n\n2. **Request Batching**\n   - Get multiple stocks in single call\n   - Uses `?symbols=AAPL,MSFT,GOOGL` format\n\n3. **Provider Fallback**\n   - Automatically tries alternative providers\n   - Prevents service interruption\n   - Graceful degradation to simulated data\n\n### Best Practices\n\n```typescript\n// ✅ Good: Batch requests\nconst stocks = await marketDataService.getMultipleStocks(\n  ['AAPL', 'MSFT', 'GOOGL', 'AMZN']\n);\n\n// ❌ Avoid: Multiple individual requests\nconst stocks = await Promise.all([\n  marketDataService.getStockQuote('AAPL'),\n  marketDataService.getStockQuote('MSFT'),\n  marketDataService.getStockQuote('GOOGL'),\n]);\n\n// ✅ Good: Use caching\n// Subsequent calls within 5 minutes return cached data\nconst stock1 = await marketDataService.getStockQuote('AAPL');\n// ... 2 minutes later\nconst stock2 = await marketDataService.getStockQuote('AAPL'); // From cache\n```\n\n## Troubleshooting\n\n### Issue: \"API Key not configured\"\n\n**Solution:**\n1. Check `.env.local` has correct variable name\n2. Ensure file is in project root\n3. Restart dev server after adding key\n4. Verify key is valid from provider's dashboard\n\n### Issue: \"Rate limit exceeded\"\n\n**Solutions:**\n1. Reduce `MAX_STOCKS_PER_REQUEST` in `.env.local`\n2. Increase `API_CACHE_DURATION` for longer caching\n3. Switch to hybrid mode to reduce API calls\n4. Upgrade to paid API tier\n\n### Issue: \"Provider failed, falling back to simulated\"\n\n**Causes:**\n- API service is down\n- Invalid API key\n- Rate limit exceeded\n- Network connectivity issue\n\n**Solutions:**\n1. Verify API status on provider's website\n2. Check API key in `.env.local`\n3. Wait 1-2 minutes for rate limit reset\n4. Check internet connection\n\n### Issue: \"Slow startup/loading times\"\n\n**Solutions:**\n1. Switch to `simulated` mode for quick testing\n2. Use `hybrid` mode to fetch fewer live stocks\n3. Reduce number of stocks to fetch\n4. Use faster API provider (Finnhub > Alpha Vantage)\n\n## Deployment\n\n### Vercel Deployment\n\n1.
